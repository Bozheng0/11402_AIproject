# 物品剩餘價值分析系統

本專案是一個「物品剩餘價值分析系統」。系統會根據使用者輸入的物品狀況描述，先由深度學習模型預測一個基礎分數，再根據物品類別、使用年限、物品狀況分數與使用頻率進行規則式加減分修正，最後得到一個保留小數的物品剩餘價值分數。

本系統不是用來單純判斷二手售價，而是用來分析物品目前還剩下多少「使用價值」與「保留價值」，可應用在斷捨離、物品整理、是否保留物品的輔助判斷等情境。

## 一、專案目的

在整理物品時，使用者常常會遇到以下問題：

- 這個物品雖然舊了，但是否仍然值得保留？
- 這個物品外觀普通，但我很常使用，是否仍有價值？
- 不同類別的物品，使用年限對價值的影響是否不同？
- 物品狀況描述能不能轉換成比較客觀的分數？
- 能不能結合模型預測與人工規則，得到更合理的剩餘價值分析結果？

因此，本專案設計一套結合「深度學習文字模型」與「規則式分數修正」的方法，用來估計物品的剩餘價值。

## 三、系統整體流程

系統主要分成兩個階段。

### 第一階段：模型預測文字基礎分數

使用者先輸入物品狀況描述，例如：

```text
I like this catcher's glove, but it's old.
```

模型會根據文字描述輸出 1～5 分各自的機率，並計算出一個 `soft_score` 作為模型基礎分數。

### 第二階段：規則式加減分修正

系統接著讓使用者輸入：

1. 物品類別
2. 使用年限
3. 物品狀況分數
4. 使用頻率

然後根據規則表對模型分數進行修正。

完整流程如下：

```text
物品狀況描述
    ↓
BiLSTM + Attention 模型輸出 1～5 分機率
    ↓
計算 soft_score
    ↓
輸入物品類別
    ↓
輸入使用年限
    ↓
套用「類別 × 使用年限」修正
    ↓
輸入物品狀況分數
    ↓
套用物品狀況分數修正
    ↓
輸入使用頻率
    ↓
依照類別套用使用頻率修正
    ↓
輸出最終物品剩餘價值分數
```

最後輸出的分數會保留小數，且不限制在 1～5 分。

## 四、模型基礎分數：soft_score

模型會先輸出每個分數的機率。例如：

```text
1 分：0.2212
2 分：0.0182
3 分：0.0393
4 分：0.0547
5 分：0.6666
```

如果只取機率最高的分數，會得到：

```text
hard_score = 5
```

因為 5 分的機率最高。

但是本系統主要使用 `soft_score`，也就是用各分數機率計算加權平均：

```text
soft_score =
1 × P(1分)
+ 2 × P(2分)
+ 3 × P(3分)
+ 4 × P(4分)
+ 5 × P(5分)
```

以上面的例子：

```text
soft_score =
1×0.2212
+ 2×0.0182
+ 3×0.0393
+ 4×0.0547
+ 5×0.6666
≈ 3.93
```

因此，雖然模型最可能判斷為 5 分，但 `soft_score` 會把其他分數的可能性也考慮進來，使結果更細緻。

本系統採用 `soft_score` 的原因是：物品剩餘價值通常不是單純的高或低，而是介於多個分數之間。使用 `soft_score` 可以避免模型因為某個正面或負面詞而直接輸出過高或過低的整數分數。

## 五、最終分數公式

本系統的最終分數公式如下：

```text
final_score =
soft_score
+ 類別 × 使用年限修正
+ 物品狀況分數修正
+ 使用頻率修正
```

各項代表意義如下：

| 項目 | 說明 |
|---|---|
| soft_score | 模型根據物品狀況描述預測出的基礎分數 |
| 類別 × 使用年限修正 | 不同類別物品因使用年限造成的價值變化 |
| 物品狀況分數修正 | 使用者輸入的實際物品狀況評分 |
| 使用頻率修正 | 根據物品類別判斷使用頻率是加分或扣分 |

## 六、規則一：類別 × 使用年限修正

不同類別的物品，使用年限對剩餘價值的影響不同。因此本系統設計「類別 × 使用年限」修正表。

| 類別 | 1年內 | 1-3年 | 3-5年 | 5-8年 | 8以上 |
|---|---:|---:|---:|---:|---:|
| 家具寢具 | +0.3 | +0.1 | -0.2 | -0.6 | -1.0 |
| 家電與3C | +0.4 | +0.1 | -0.4 | -0.9 | -1.3 |
| 服飾配件 | +0.3 | 0.0 | -0.4 | -0.8 | -1.2 |
| 美妝個護 | +0.2 | -0.4 | -1.0 | -1.5 | -2.0 |
| 書籍辦公 | +0.2 | +0.1 | 0.0 | -0.2 | -0.4 |
| 生活廚具 | +0.3 | 0.0 | -0.3 | -0.7 | -1.1 |
| 運動愛好 | +0.3 | 0.0 | -0.4 | -0.8 | -1.2 |
| 紀念品 | +0.4 | +0.3 | +0.2 | +0.1 | 0.0 |

### 設計邏輯

- **家具寢具**：通常壽命較長，短期使用不一定會大幅降低價值，但使用 5 年以上可能開始出現明顯磨損。
- **家電與3C**：折舊速度較快，且可能受到電池、零件老化或規格過時影響，因此年限越久扣分越多。
- **服飾配件**：會受到穿著次數、清洗、流行程度與保存狀況影響，因此 3 年以上開始較明顯扣分。
- **美妝個護**：涉及保存期限與衛生問題，因此年限影響最大。
- **書籍辦公**：只要保存良好，年限影響較小。
- **生活廚具**：會因使用產生刮痕、污漬與衛生問題，因此年限越久扣分越多。
- **運動愛好**：運動用品常因使用而磨損，例如球拍、手套、鞋子、護具等，因此年限越久價值下降。
- **紀念品**：紀念品的價值不一定隨時間下降，因此年限修正較寬鬆。

## 七、規則二：物品狀況分數修正

使用者會輸入物品狀況分數，範圍為 1～5。

| 物品狀況分數 | 意義 | 修正分數 |
|---:|---|---:|
| 1 | 狀況很差 | -0.70 |
| 2 | 狀況偏差 | -0.35 |
| 3 | 普通 | 0.00 |
| 4 | 狀況良好 | +0.35 |
| 5 | 狀況很好 | +0.70 |

### 設計邏輯

模型是根據文字描述進行判斷，但文字描述可能不完整，也可能同時包含正面與負面資訊。

例如：

```text
Its exterior was tattered and worn, but it felt very nice to the touch.
```

這種句子同時包含：

- `tattered and worn`：破舊、磨損
- `very nice`：觸感很好

模型可能因為看到正面詞而給出偏高分數，因此加入使用者自行輸入的物品狀況分數，讓實際狀況可以參與最後評分。

## 八、規則三：使用頻率修正

使用者會輸入使用頻率，範圍為 1～5。

| 使用頻率 | 意義 |
|---:|---|
| 1 | 幾乎沒使用 |
| 2 | 偶爾使用 |
| 3 | 普通使用 |
| 4 | 經常使用 |
| 5 | 非常常使用 |

使用頻率的修正會依照物品類別不同而改變。本系統將物品分成兩種情況：

1. 使用越頻繁，代表磨損越高，因此分數下降。
2. 使用越頻繁，代表仍有實用價值，因此分數上升。

### 1. 容易因使用而磨損的類別

以下類別使用越頻繁，代表磨損可能越高，因此使用頻率越高，分數越低：

```text
家具寢具
家電與3C
生活廚具
運動愛好
```

| 使用頻率 | 修正分數 |
|---:|---:|
| 1 | +0.30 |
| 2 | +0.10 |
| 3 | 0.00 |
| 4 | -0.30 |
| 5 | -0.60 |

### 設計邏輯

這些類別通常會因為頻繁使用而產生磨損或耗損。

例如：

- **家具寢具**：可能出現坐墊塌陷、床墊變形、布料磨損等問題。
- **家電與3C**：可能出現電池老化、零件耗損、效能下降或規格過時等問題。
- **生活廚具**：可能出現刮痕、油污、變色、衛生疑慮等問題。
- **運動愛好**：可能出現球鞋磨損、手套老化、護具彈性下降、球拍損耗等問題。

因此，對這些類別來說，使用頻率越高，代表物品被消耗的程度可能越大，所以會降低剩餘價值分數。

### 2. 其他類別

其他類別使用越頻繁，代表物品對使用者仍然有實用性或保留價值，因此使用頻率越高，分數越高。

| 使用頻率 | 修正分數 |
|---:|---:|
| 1 | -0.20 |
| 2 | 0.00 |
| 3 | +0.20 |
| 4 | +0.40 |
| 5 | +0.60 |

### 設計邏輯

對於部分物品而言，使用頻率高不一定代表價值下降，反而可能表示它仍然對使用者有實際功能或情感價值。

例如：

- **書籍辦公**：常使用可能代表內容仍有參考價值或學習價值。
- **服飾配件**：若仍常使用，可能代表符合需求或仍具搭配價值。
- **美妝個護**：若仍常使用，可能代表使用者仍有需求，但仍需搭配使用年限與狀況分數判斷。
- **紀念品**：使用或觀看頻率高，可能代表情感價值較高。

因此，對非高度磨損型類別來說，使用頻率越高，會被視為仍有保留價值，因此給予加分。

## 九、完整計算範例

本系統的最終分數計算公式為：

```text
final_score =
soft_score
+ 類別 × 使用年限修正
+ 物品狀況分數修正
+ 使用頻率修正
```

其中：

- `soft_score` 是模型根據物品描述預測出的基礎分數。
- `類別 × 使用年限修正` 是根據物品類別與使用年限查表得到的修正值。
- `物品狀況分數修正` 是根據使用者輸入的物品狀況分數得到的修正值。
- `使用頻率修正` 是根據物品類別與使用頻率得到的修正值。

### 範例一：運動用品

假設使用者輸入的物品描述為：

```text
I like this catcher's glove, but it's old.
```

模型根據描述計算出：

```text
soft_score = 4.0134
```

使用者接著輸入：

```text
類別：運動愛好
使用年限：1-3年
物品狀況分數：3
使用頻率：5
```

根據規則表：

```text
類別 × 使用年限修正 = 0.0
物品狀況分數修正 = 0.0
使用頻率修正 = -0.6
```

因此：

```text
final_score = 4.0134 + 0.0 + 0.0 - 0.6
final_score = 3.4134
```

最後輸出：

```text
最後小數分數：3.41
```

此範例中，雖然模型根據文字描述判斷物品仍有一定價值，但因為「運動愛好」屬於容易磨損的類別，而且使用頻率為 5，代表非常常使用，因此會扣除 0.6 分。

### 範例二：服飾配件

假設模型根據物品描述得到：

```text
soft_score = 3.9272
```

使用者輸入：

```text
類別：服飾配件
使用年限：1-3年
物品狀況分數：4
使用頻率：1
```

根據規則表：

```text
類別 × 使用年限修正 = 0.0
物品狀況分數修正 = +0.35
使用頻率修正 = -0.20
```

因此：

```text
final_score = 3.9272 + 0.0 + 0.35 - 0.20
final_score = 4.0772
```

最後輸出：

```text
最後小數分數：4.08
```

此範例中，物品狀況分數為 4，代表使用者認為物品狀況良好，因此加 0.35 分。但使用頻率為 1，代表幾乎沒使用，對非磨損型類別來說保留價值較低，因此扣 0.20 分。

### 範例三：家電與3C

假設模型根據物品描述得到：

```text
soft_score = 4.5000
```

使用者輸入：

```text
類別：家電與3C
使用年限：3-5年
物品狀況分數：4
使用頻率：5
```

根據規則表：

```text
類別 × 使用年限修正 = -0.4
物品狀況分數修正 = +0.35
使用頻率修正 = -0.6
```

因此：

```text
final_score = 4.5000 - 0.4 + 0.35 - 0.6
final_score = 3.8500
```

最後輸出：

```text
最後小數分數：3.85
```

此範例中，家電與3C會因使用年限與高頻率使用產生折舊，因此即使物品狀況分數為 4，最後仍會被修正到 3.85 分。

### 範例四：紀念品

假設模型根據物品描述得到：

```text
soft_score = 3.8000
```

使用者輸入：

```text
類別：紀念品
使用年限：8以上
物品狀況分數：5
使用頻率：5
```

根據規則表：

```text
類別 × 使用年限修正 = 0.0
物品狀況分數修正 = +0.70
使用頻率修正 = +0.60
```

因此：

```text
final_score = 3.8000 + 0.0 + 0.70 + 0.60
final_score = 5.1000
```

最後輸出：

```text
最後小數分數：5.10
```

此範例中，紀念品不一定會因年限增加而降低價值，且使用頻率高可能代表情感價值或保留價值較高，因此最後分數可以高於 5 分。

### 小結

透過上述規則，本系統不只依賴模型根據文字描述做出的判斷，也會進一步考慮物品的實際使用條件。

因此，最終分數能同時反映：

1. 文字描述中的物品狀況。
2. 不同類別的折舊特性。
3. 使用年限造成的價值變化。
4. 使用者主觀輸入的物品狀況分數。
5. 使用頻率對剩餘價值的影響。

## 十、模型訓練設定

模型使用 BiLSTM + Attention 架構，並以 Validation Macro-F1 作為最佳模型選擇標準。

目前主要訓練設定如下：

```text
EPOCHS = 5
loss_function = CrossEntropyLoss(label_smoothing=0.05)
selection_metric = valid_macro_f1
use_weighted_sampler = True
use_class_weight = False
```

### 為什麼使用 Macro-F1？

資料中的高分樣本較多，如果只看 Accuracy，模型可能傾向預測高分。Macro-F1 會平均考慮 1～5 分各類別的表現，因此更適合用來觀察模型是否只偏向某個分數。

### 為什麼使用 label smoothing？

評分任務本身具有模糊性。例如 4 分和 5 分之間可能沒有非常明確的界線。`label_smoothing=0.05` 可以降低模型過度自信的情況，使模型預測較穩定。
